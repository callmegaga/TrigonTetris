# js/ts 规范

## 命名规范

### 全局变量

- 命名方法：所有字母大写与下划线。
- 命名规范：单词之间用下划线连接。
- 命名建议：
  - 尽量在变量名字中体现所属类型，如:length、count等表示数字类型；而包含name、title表示为字符串类型。
  - 赋值在等号左右保留一个空格
  - 冒号右边应该保留一个空格
  - 数组的多个元素，应该间隔一个空格

### 局部变量

- 命名方法：所有字母小写与下划线。
- 命名规范：单词之间用下划线连接。
- 命名建议：
  - 尽量在变量名字中体现所属类型，如:length、count等表示数字类型；而包含name、title表示为字符串类型。

### 模块文件

因为在模块文件中，不存在传统意义上的全局变量。

所以变量都以字母小写与下划线的形式命名

常量以大写字母与下划线的形式命名

在能使用常量的情况下，需要使用常量

### 函数

- 命名方法：驼峰命名，首字母小写。
- 命名规范：首个单词应为动词，如:get、remove等。
- 命名建议：
  - 尽量在变量名字中体现函数的行为，如getPersonInfo、removeBranch。
  - 函数名和左括号之间保留一个空格

### 类成员

- 公共属性和方法：跟局部变量和函数的命名一样。
- 私有属性和方法：前缀为\_(下划线)，后面跟公共属性和方法一样的命名方式。

### 注释规范

#### 单行注释

- 说明：单行注释以两个斜线开始，以行尾结束。
- 语法：// 这是单行注释
- 使用方式：
  - 单独一行：//(双斜线)与注释文字之间保留一个空格。
  - 在代码后面添加注释：//(双斜线)与代码之间保留一个空格，并且//(双斜线)与注释文字之间保留一个空格。
  - 注释代码：//(双斜线)与代码之间保留一个空格。

#### 多行注释

- 说明：以/\*开头，\*/结尾
- 语法：/_ 注释说明 _/
- 使用方法：
  - 若开始(/\*)和结束(\*/)都在一行，推荐采用单行注释。
  - 若至少三行注释时，第一行为/\*，最后行为\*/，其他行以空格开始。

### 代码规范

- 使用tag进行缩进
- 不能省略分号
- if、switch等与条件语句的括号保留一个空格
- 操作符左右保留空格
- 字符串默认使用双引号，只有在双引号会造成额外转义或模板字符串更合适时，才使用其它写法
