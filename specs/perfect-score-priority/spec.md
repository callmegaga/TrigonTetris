# Spec

## 基本信息

- 功能名称：Perfect 按得分最高优先
- 对应 Proposal：`specs/perfect-score-priority/proposal.md`
- 当前状态：approved
- 最后更新：2026-05-20

## 问题定义

正常落地后同时存在多个 `Perfect Square` / `Perfect BevelledSquare` 时，系统需要选择当前可获得分数最高的候选，而不是固定让斜正方形优先于普通正方形。

## 用户故事

- 作为：游戏玩家
- 我希望：同时形成多个 Perfect 时优先清除分数最高的那个
- 从而：高价值拼法不会被低价值拼法抢先消除

## 功能行为

### 主流程

1. 活动块正常落地，并且没有进入 `ExtendLife`。
2. 系统扫描棋盘上所有合法 Perfect 普通正方形和斜正方形。
3. 系统用当前 `calculateSquareScore(boards, square, true)` 计算每个候选得分。
4. 系统选择得分最高的候选并触发现有 Perfect 消除流程。
5. 如果没有任何 Perfect 候选，继续现有绘制和下一轮下落流程。

### 平分规则

1. 得分不同：分数高者优先。
2. 得分相同：沿用现有几何优先级，面积大者优先。
3. 得分和面积都相同：更靠下者优先。
4. 得分、面积、位置都相同：保持扫描得到的先选结果，不额外引入随机性。

### 异常流程

1. 如果棋盘无合法 Perfect 候选，返回 `undefined`。
2. 如果候选得分为 0 或负数，仍按统一比较处理；当前公式下合法 Perfect 不应产生负分。

### 空状态/加载态/错误态

- 加载态：不涉及
- 空状态：无 Perfect 候选时不触发消除
- 错误态：不涉及
- 重试机制：不涉及

## 验收标准

- [ ] 正常落地后，如果普通正方形得分高于斜正方形，优先选择普通正方形。
- [ ] 正常落地后，如果斜正方形得分高于普通正方形，优先选择斜正方形。
- [ ] 得分相同时仍按面积、更靠下的顺序稳定选择。
- [ ] `Cover` 优先级保持现状。
- [ ] `MoveBoard` 阶段仍只扫描普通 Perfect。
- [ ] focused unit test、lint、format、build 通过。

## 非功能要求

- 性能：候选扫描发生在落地判定时，棋盘规模固定，允许一次性扫描普通和斜正方形候选。
- 可维护性：统一选择逻辑放在 rule 层，`Game` 只调用一个入口。
- 可测试性：选择函数必须可被单元测试直接覆盖。
