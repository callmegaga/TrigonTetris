# Spec

## 基本信息

- 功能名称：Perfect 清除后仅保留横向 SpreadLight
- 对应 Proposal：`specs/horizontal-spread-light/proposal.md`
- 当前状态：approved
- 最后更新：2026-05-08

## 问题定义

`Perfect` 图形清除后的 SpreadLight 需要从“上下左右四向扩散并清除”调整为“仅左右横向扩散并清除”，避免视觉表现和实际清除范围不一致。

## 用户故事

- 作为：玩家
- 我希望：Perfect 后只触发横向光芒清除
- 从而：更清楚地预期本次消除会影响哪些 block

## 功能行为

### 主流程

1. 游戏在 `Active` 或 `MoveBoard` 后续扫描中命中 `Perfect Square` 或 `Perfect BevelledSquare`。
2. 系统先清除组成该 `Perfect` 图形的 block。
3. 系统播放组成图形的消除动画。
4. 系统播放左右横向 SpreadLight 动画。
5. 系统只收集横向光带覆盖到的剩余 block。
6. 系统清除这些横向光带 block，并从 `dead_blocks` 移除。
7. 系统播放横向光带 block 的消除动画。
8. 清除结束后进入 `MoveBoard`。

### 异常流程

1. 如果横向光带内没有剩余 block，系统仍应完成动画并进入 `MoveBoard`。
2. 如果命中的是 `Perfect BevelledSquare`，横向光带按其外接轴对齐正方形的高度计算。

### 空状态/加载态/错误态

- 加载态：不涉及。
- 空状态：横向光带没有 block 时，清除集合为空。
- 错误态：不新增错误态。
- 重试机制：不涉及。

## 验收标准

- [ ] `Perfect Square` 后只清除同高横向光带中的 block。
- [ ] 位于同列但不在横向光带行范围内的 block 不被清除。
- [ ] `Perfect BevelledSquare` 的横向光带按外接轴对齐正方形计算。
- [ ] SpreadLight 视觉动画只向左右扩散，不再向上下扩散。
- [ ] 相关单元测试通过。
- [ ] `pnpm build` 通过。
