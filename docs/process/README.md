# 开发流程说明

本目录用于定义项目的标准交付流程，保证人和 AI 采用同一套阶段、门槛和证据要求开展开发。

## 建议阅读顺序

1. `feature-lifecycle.md`：说明一个功能从需求进入到验收收尾的标准阶段和出口条件
2. `ai-collaboration-policy.md`：说明 AI 在项目中的职责边界、输入要求和输出约束
3. `multi-agent-collaboration-contract.md`：说明多人和多 AI agent 协作时的 owner、写入范围、交接和冲突处理
4. `spec-review-checklist.md`：说明评审规格时要检查的完整性、可执行性和可验证性
5. `definition-of-done.md`：说明一个功能在规格、实现、质量和验收上满足什么条件才算完成

## AI 开发必备能力

- `Context7`：用于查询框架、库和工具链的最新官方文档与推荐用法，特别适用于初始化、安装、升级、迁移等工作
- `superpowers`：用于增强 AI 在项目中的协作、规范执行和工作流能力，作为通用 AI 开发增强能力；若当前工具支持，在需求梳理和生成 `proposal/spec` 阶段应优先使用它进行需求逻辑梳理

若运行环境暂时不具备上述能力，AI 必须主动说明，并以“查询最新官方文档并给出来源链接”的方式执行替代流程，而不是凭经验直接安装或升级依赖。

## 目标

- 让需求进入实现前具备足够清晰的规格
- 让 AI 的输出有明确边界和质量要求
- 让每次交付都有可追踪的验证证据
