# Spec

## 基本信息

- 功能名称：Perfect 得分公式调整
- 对应 Proposal：`specs/perfect-score-formula/proposal.md`
- 当前状态：approved
- 最后更新：2026-05-08

## 问题定义

系统需要将 `Perfect` 得分公式统一为 `面积 * 板块个数 * 10^(颜色数 - 2)`，并继续保持 `Cover` 不得分。

## 用户故事

- 作为：玩家
- 我希望：普通正方形和斜正方形使用统一得分公式
- 从而：能更直观地理解不同拼法为什么得到不同分数

## 功能行为

### 主流程

1. 游戏命中 `Perfect Square` 或 `Perfect BevelledSquare`。
2. 系统收集该图形参与的 block 数量与颜色数量。
3. 系统计算图形面积。
4. 系统按 `面积 * 板块个数 * 10^(颜色数 - 2)` 得出本次 gain。
5. UI 按既有流程累计当前分数、展示加分浮层并更新历史最高分。

### 异常流程

1. 如果是 `Cover`，得分固定为 0。
2. 如果颜色数量为 2，则颜色倍率为 1。

## 验收标准

- [ ] 普通正方形得分为 `size * size * blockCount * 10^(colorCount - 2)`。
- [ ] 斜正方形得分面积为外接横竖正方形面积，即 `4 * size * size`。
- [ ] `Cover` 得分仍为 0。
- [ ] 示例分数与新公式一致。
- [ ] 单元测试通过。
- [ ] `pnpm build` 通过。
